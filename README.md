CS 210 Project Reflection

For this portfolio entry, I chose to include my Airgead Banking project from CS 210. This program was designed to calculate investment growth based on user‑entered data, and it helped automate a process that would normally require manual math and repeated calculations. The goal was to build a functional C++ program that could handle input, perform calculations, and display results in a clear and organized way.

One thing I feel I did well in this project was keeping my code organized and readable. I used functions to break the program into smaller pieces, which made it easier to understand and test. I also added comments throughout the code so someone else could follow my logic without getting lost.

If I were to improve the project, I would focus on adding stronger input validation and making the program more flexible for different types of financial scenarios. These changes would make the program more secure and more useful for real‑world situations. I would also look for ways to simplify some of the calculations to make the code more efficient.

The most challenging part of this project was debugging the logic behind the compound interest calculations. At first, my numbers weren’t matching what I expected. I fixed this by stepping through the code slowly, printing out values, and reviewing the formulas. I also used online documentation and course resources to double‑check my approach. These tools have become part of my support network for future programming work.

This project helped me build several skills that I can use in other classes and future projects. I strengthened my understanding of C++ syntax, functions, loops, and user input. I also practiced problem‑solving and debugging, which are skills that transfer to any programming language.

I made the program maintainable and adaptable by using clear variable names, consistent formatting, and separating the logic into functions. This structure makes it easier to update the program later or add new features without rewriting everything.
